# Corpus Synthesis

## NORMALIZE BEFORE COMPARING

Map source-specific scenes to shared communication jobs before comparing visual forms:

- establish identity or atmosphere;
- orient the visitor;
- state an argument;
- reveal proof;
- sequence projects or products;
- explain process or authority;
- transition between routes or chapters;
- support inquiry or closure.

Compare a circle, image expansion, fixed metadata field, and panel transfer only when they perform comparable jobs. Do not group mechanics merely because they move similarly.

## CORPUS MAP

Produce:

1. **Recurring patterns** — supported by multiple independent sources.
2. **Unique mechanics** — source-specific behavior worth examining without treating novelty as superiority.
3. **Negative patterns** — repeated approaches that conflict with the active brand or constraints.
4. **Strong evidence** — repeated, reversible, pointer-confirmed, or demonstrated across routes.
5. **Weak evidence** — trigger-ambiguous, mediated, partial, or dependent on one interpretation.
6. **Contradictions** — sources or reports describing incompatible states, labels, or mechanics.

Use a matrix:

| Pattern | Communication Job | Sources | Strongest Evidence | Cost / Risk | Brand Decision Pending |
| --- | --- | --- | --- | --- | --- |

## EVIDENCE RANKING

Rank source usefulness by:

- chronology completeness;
- direct interaction evidence;
- reverse behavior;
- repeated demonstrations;
- text and state legibility;
- observation/inference discipline;
- ambiguity disclosure;
- relevance to the named questions.

Do not rank the visual design itself. Rank what the source can prove.

## CONTRADICTION PROTOCOL

For every contradiction:

1. Quote or summarize both claims without merging them.
2. Check whether they describe different pages, breakpoints, states, or moments.
3. Prefer direct and repeated evidence over mediated or single-instance evidence.
4. Preserve unresolved ambiguity when reconciliation would require guessing.
5. Prevent ambiguous mechanics from becoming expensive implementation commitments.

## BIAS AUDIT

Run these checks before recommending patterns:

| Bias | Diagnostic | Correction |
| --- | --- | --- |
| Selection bias | Were references chosen only because they already resemble the desired answer? | Include contrasting and restrained examples or state the limitation |
| Prestige bias | Would the mechanic still appear useful without awards, fame, or polish? | Apply the removability test |
| Novelty bias | Is uniqueness being mistaken for relevance? | Score communication value before distinctiveness |
| Polish imbalance | Is one territory winning because its reference is better produced? | Compare underlying logic at equal imagined finish |
| Frequency bias | Is a recurring pattern assumed correct merely because it is common? | Test it against brand truth and constraints |
| Survivorship bias | Are production failures and maintenance costs invisible? | Add asset, performance, responsive, and operational burdens |
| Confirmation bias | Were rejection reasons sought only for disliked options? | Steel-man every serious candidate before deciding |

## PATTERN CONCENTRATION

Count signature mechanics by chapter and across the page. Flag:

- more than one dominant mechanic competing in a chapter;
- the same effect repeated without a shared identity reason;
- several references contributing conspicuous forms to one screen;
- motion intensity that never yields to reading;
- a reference whose section architecture is being copied wholesale.

Recommend a hierarchy: one governing system, one or few signature events, quiet supporting behavior, and deliberate stillness.

## SYNTHESIS EXIT GATE

Advance to brand translation only when:

- source coverage answers the named questions or exposes the gap;
- strongest and weakest evidence are visible;
- contradictions remain explicit;
- recurring and unique mechanics are separated;
- prestige and selection bias have been tested;
- no implementation technology is presented as observed without confirmation.
