# Nested Narrative Labyrinth

**Source:** Cloud Atlas • David Mitchell • 2004 • Literary Fiction
**Motion personality:** Liquid
**Density:** Maximalist
**Psychological lever:** Curiosity gap + pattern-matching compulsion

---

## How It Works

Interleave multiple stories from different eras without explaining connections.
The user must assemble the pattern themselves. The pleasure is in the puzzle.

**In the book:** Six stories spanning centuries, each interrupted at a cliffhanger.
Stories are nested like Russian dolls — first five told first half, then resolved
in reverse order. Connections emerge through theme, not plot.

**In web design:** Scroll-triggered section transitions with parallel content
streams that converge. The connections emerge through design, not explanation.

---

## Web Translation

**Hero / above-the-fold:**
- Stacked "story cards" — each one a different era, genre, visual style
- Headline: "All lives intersect."
- The connections are withheld

**Spatial / 3D / canvas:**
- Multiple interlocking planes in R3F — each plane is a "story"
- Camera starts zoomed in on one plane, gradually zooms out to reveal entire structure
- Planes connect through shared vertices or particle bridges

**Scroll pacing:**
- 1.5x viewport per story section
- sine.inOut easing
- Micro-animation: each story has its own visual language (color, typography, motion)
- But shares a common "thread" element that connects them

**Micro-copy & CTA:**
- CTA appears only after all stories are visible
- Example: "See the connections" — center-screen at convergence point
