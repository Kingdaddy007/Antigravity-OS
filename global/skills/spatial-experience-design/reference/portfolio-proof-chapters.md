# Portfolio Proof Chapters

Use this when planning interior designer portfolios, project galleries, case studies, hover previews, or project-led cinematic websites. A portfolio must prove judgment before it becomes a grid.

## Core Rule

Do not show beautiful projects as interchangeable cards. Show why the designer's decisions were necessary.

## Portfolio Journey

Use:

`Curated chapter -> Decision proof -> Material/process evidence -> Project index -> Inquiry relevance`

Lead with one strong cinematic project chapter before broad scanning. A grid can follow, but only after the visitor has felt taste and proof.

## Project Chapter Schema

```text
Project:
Client/context:
Room or space:
Original problem:
Design decision:
Material/light move:
Before/after or transformation proof:
Cinematic media:
Caption/proof text:
Interaction:
Inquiry relevance:
```

## Proof Types

| Proof Type | Use For | Presentation |
| --- | --- | --- |
| Before/after | Staging, renovation, redesign | Reveal with decision captions |
| Material/process | Craft-led studios | Macro texture, sourcing, sketches, samples |
| Room sequence | Full-service residential work | Chaptered rooms with emotional pacing |
| Constraint proof | Complex client/site problems | Problem -> decision -> result |
| Founder eye | Founder-led taste brands | Selection, rejection, arrangement, refinement |
| Press/client proof | Premium authority | Quiet citation, not logo clutter |

## Interaction Patterns

- Hover/tap reveals alternate image, material note, or decision caption.
- Project index preview shows one strong image, not a collage dump.
- Shared-element transition can expand a project plate into a chapter.
- Before/after controls must caption the design decision, not only the result.
- Mobile interaction must become tap or stacked chapter, not hover-only.

## Anti-Patterns

| Anti-Pattern | What It Is | Fix |
| --- | --- | --- |
| Portfolio Dump | Many equal cards above proof | Lead with one curated chapter |
| Pretty No Argument | Images without decisions | Add problem, move, material, result |
| Template Case Study | Same layout for every project | Let each project's proof type shape the chapter |
| Hover Only | Desktop-only proof | Add tap, keyboard, and mobile alternatives |
| Fake Metrics | Decorative stats | Use real project facts or omit |

## Output

Create `contexts/spatial/portfolio-proof-chapters.md` when portfolio is central. Each chapter must name the proof job, visual media, interaction, and inquiry relevance.
