# Engineering Baseline

- Detect and respect the active project's language, versions, framework, package manager, and conventions.
- Treat TypeScript, native compilers, motion libraries, and other stack choices as conditional preferences, never universal mandates.
- Reduce scope before reducing correctness, security, accessibility, or necessary verification.
- Keep diagnostics read-only unless implementation was requested.
- Verify observable behavior and relevant failure paths before declaring completion.
- Prefer portable capability descriptions over vendor-specific tool names in canonical content.
