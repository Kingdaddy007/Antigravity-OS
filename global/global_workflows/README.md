# Workflows

This review workspace routes visual/frontend work through a spatial experience system for **high-end interior storytelling websites**.

Old broad visual workflows were preserved in `_archive-generic-visual-system/`.

## Active Spatial Workflow Spine

1. `workflow-project-inception.md` - start a spatial website project and create the north star contexts.
2. `workflow-spatial-concept.md` - required pre-build creative contract.
3. `workflow-visual-brainstorm.md` - explores directions only after spatial artifacts exist.
4. `workflow-design-ui.md` - routes spatial UI work through the correct path.
5. `workflow-impeccable-craft.md` - builds only after the spatial contract exists.
6. `workflow-impeccable-animate.md` - plans and implements arrival, ambient, scroll-bound, and interaction motion.

## Rule

Do not build a spatial brand page from a generic hero/services/portfolio/testimonials/contact sequence. The default journey is:

`Atmosphere -> Taste -> Transformation -> Proof -> Method -> Inquiry`

Beloved is part of the creative system. Use concept-forge questions when the direction is unclear, propose mixed concepts, cite `design-audit/` mechanics, then produce Beloved's Asset Directive before build.
