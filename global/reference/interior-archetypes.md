# Interior Archetypes

Use one primary archetype per spatial website. Borrow from one secondary only when it supports the same visual thesis.

## Gallery Procession

Use for interior decorator portfolios, spatial studios, editorial residences, and luxury project showcases.

- Structure: entry room -> hero project -> transformation/proof -> method -> inquiry.
- Composition: one focal room/image per viewport, generous margins, quiet captions.
- Motion: slow plate reveals, stacked media panels, room color/light thresholds.
- UI: peripheral navigation, tiny metadata, concierge CTA.
- Avoid: multi-card grids above the fold, fast carousels, loud buttons.

## Layered Diorama

Use for immersive first folds and object/material-led brands.

- Structure: atmosphere layer, display type, midground room/object, proof/UI, foreground blur.
- Assets: transparent PNG/WebP layers or rendered elements with matching light/perspective.
- Motion: arrival timeline plus ambient drift plus mouse/scroll parallax.
- Avoid: CSS-only fake furniture, plants, rooms, murals, or depth.

## Signature Object Spine

Use when one spatial object can carry the brand: lamp, chair, vase, doorway, mural, fabric roll, table, material sample, staircase.

- Structure: object appears in hero, guides chapters, becomes proof/annotation/transition.
- Required: object role inventory and object blocking map.
- Motion: object wipe, annotation projection, slow inspection, camera push, scroll path.
- Avoid: decorative 3D with no structural consequence.

## Editorial / Invisible Luxury

Use for ultra-premium decorators, private residences, estates, galleries, and high-ticket inquiry sites.

- Structure: full-bleed chapters, sparse copy, delayed proof, quiet inquiry.
- UI: borderless nav, no loud CTAs, understated metadata.
- Motion: slow stacked panels, focus pulls, threshold cuts.
- Avoid: service packages, listing grids, search filters, generic lead-gen forms above the fold.

## Transformation Reveal

Use for staging, redesign, renovation, styling, and before/after proof.

- Structure: problem scene -> intervention -> transformed room -> decision captions -> proof.
- Motion: curtain split, light sweep, slider, plan-to-room, material wipe.
- UI: captions explain decisions, constraints, and outcome.
- Avoid: before/after as a novelty widget without story.

## Material Library Story

Use when the studio's authority is in sourcing, finishes, furniture, textiles, stone, plaster, lighting, or craft.

- Structure: macro material entry -> room application -> sourcing/proof -> method.
- Motion: tactile closeups, swatch reveals, grazing light, texture transitions.
- UI: swatches with names, finishes, room roles, and use logic.
- Avoid: color dots without material meaning.
